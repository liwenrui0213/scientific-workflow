# Alignment and Claim Cases

Read this reference only when an intake ambiguity could reasonably be blocking,
deferred, or safely provisional, or when a proposed Claim is difficult to make
falsifiable without changing the human's intent.

## Classify the decision, not the topic

The same topic can fall into different classes depending on the next authorized
action.

### Ask now

Ask when plausible answers authorize materially different research or immediate
irreversible work and no conservative reversible default exists.

- “Improve accuracy” could mean energy error, force error, or phase
  classification, and the requested Study would otherwise approve an expensive
  benchmark immediately.
- The idea says “keep symmetry” but does not identify whether the protected
  property is exact equivariance, approximate covariance, or only invariant
  outputs.
- Two data splits encode scientifically different extrapolation questions and
  the next action would commit one as the evaluation authority.

State the current interpretation, the alternatives, and why the immediate gate
cannot be crossed without the answer. Ask no more than three independent
questions in one batch.

### Defer to the consequential boundary

Record the question without blocking intake when it matters later.

- GPU model matters for a performance Cohort but no performance Run is being
  authorized yet.
- A solver tolerance matters before a production protocol or Evidence, while
  low-cost exploratory derivations can proceed.
- The exact ablation matrix matters after candidate mechanisms are known.

Name the future boundary at which the question must be resolved.

### Continue provisionally

Use a stated conservative assumption when it is cheap to reverse and cannot
change protected scientific meaning.

- Select a disposable notebook filename under `work/active/`.
- Inspect two plausible source modules before deciding which implementation path
  is relevant.
- Use a tiny synthetic case for debugging while explicitly withholding Evidence
  status.

## Draft falsifiable Claims

A proposed Claim should identify:

1. the object or method under study;
2. the comparison or predicted relationship;
3. the quantities observed;
4. the conditions and scope;
5. the evidence that could contradict it.

Prefer:

> Within the declared discretization and parameter range, method A has lower
> maximum residual than baseline B at equal compute budget, without violating
> invariant I.

Here `I` denotes the invariant explicitly defined in the Brief or model.

Avoid:

> Method A is better.

Do not fabricate numerical thresholds to make a Claim look precise. If the human
did not authorize a threshold, record it as unresolved or defer it to evaluator
formalization.

## Anti-patterns

- Asking for choices that repository inspection can answer.
- Turning every missing implementation detail into an intake blocker.
- Treating an omitted budget as unlimited permission.
- Hiding an Agent inference as a human assumption.
- Asking repeated variants of the same unresolved question.
- Expanding the requested scientific question while “making it precise.”
